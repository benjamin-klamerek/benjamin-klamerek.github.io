## Flash tooling

### Device accessibility

Exactly the same as for OpenBeken.

![etersky.jpg]({% link IOT_projects/roller_shutters/images/espburner.jpg %})

Previously, it seemed that you could flash directly with OTA, but that is no longer the case.

### Tool

Tasmota provides a very useful tool to flash devices : https://tasmota.github.io/install/

It is very convenient because you can directly set up your Wi-Fi parameters.

But, as always, we will have to do things differently.

In my case, I compiled the Tasmota project myself because I wanted to enable [blinds and shutters](https://tasmota.github.io/docs/Blinds-and-Shutters/).

With VS Code and PlatformIO, setting up the project is quite straightforward.

You will need to update *user_config_override.h* to define the Wi-Fi configuration, MQTT broker, and to enable the shutter code:

(https://tasmota.github.io/docs/Compile-your-build/#enabling-a-feature-in-tasmota
)
```
#ifndef USE_SHUTTER
#define USE_SHUTTER           // Add Shutter support (+6k code)
#endif
```

{: .note }
You may encounter issues with *firmware.bin* due to insufficient space. Use the compressed version (automatically generated by the project).
Therefore, you may need to install the minimal version before flashing your custom version. Again, due to size constraints.
